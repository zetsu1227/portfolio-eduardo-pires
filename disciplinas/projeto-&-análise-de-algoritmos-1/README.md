# 🧩 < Projeto e Análise de Algoritmos 1 >
**Período:** 2025/2  
**Projeto:** Manipulação de Mapas  
**Professor:** Bruno Iochins Grisci e Rodrigo Machado

---

## 🎯 Problema a ser resolvido
Nesta cadeira, aprendemos sobre entender a complexidade de algoritmos, utilização de grafos e lógica gulosa.
Ao longo do semestre, os professores passaram vários exercícios do Beecrowd para fazer. Nessa sentido, resolvi fazer
um projeto para colocar em prática os principais temas que aprendi. Esse projeto envolve em manipular mapas por meio
de grafos e com isso usar algoritmos para encontrar caminhos/trilhas a fim de analisar o seu funcionamento.

---

## 🏗️ Arquitetura e Tecnologias

- A escolha da linguagem era livre, eu optei por C++
- Beecrowd como plataforma de resolução de exercícios
- Visual Studio 2022 como ambiente de desenvolvimento
- Arquitetura baseada em Tipo Abstrato de Dados(TAD) com o objetivo de organizar as funções e estruturas

---

## 🧱 Boas práticas aplicadas

- Versionamento e fluxo de desenvolvimento por meio do GitHub Desktop
- Uso de TADs e comentários nos códigos
- Modularidade das funções e da main

## 📈 Resultados e Aprendizados
- O que funcionou bem:  
  1. A adoção de comentários claros e padronizados contribuiu para a legibilidade do código e para a documentação das decisões de implementação.
  2. O uso do GitHub Desktop para controle de versões permitiu acompanhar a evolução do projeto, testar alterações de forma segura e recuperar versões anteriores quando necessário.
  
- O que poderia melhorar:
  1. A implementação do algoritmo de Prim poderia ser otimizada com o uso de uma fila de prioridade(min-heap), reduzindo a complexidade de tempo em grafos grandes.
  2. Disponibilidade de uma pasta contendo testes com grafos maiores e casos extremos, que poderiam aumentar a robustez da solução.
  
- Conceitos mais aplicados da disciplina:  
  1. Compreensão dos tipos de grafos e seus algoritmos.
  2. Identificação de soluções gulosas.
  3. Análise de complexidade de tempo de algoritmos.

- Lições para projetos futuros:  
  1. Avaliar previamente diferentes estruturas de dados para escolher a que oferece melhor desempenho conforme o tamanho e a densidade do grafo.
  2. Continuar priorizando legibilidade e documentação, especialmente pensando em projetos colaborativos ou de longo prazo.
