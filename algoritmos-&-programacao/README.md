# 🧩 <Algoritmos & Programação>
**Período:** 2025/1  
**Projeto:** Jogo 2D  
**Professor:** Renan de Queiroz Maffei

---

## 🎯 Problema a ser resolvido
Inicialmente, esta cadeira introduziu inúmeros conceitos de programação para mim. 
Sabendo disso, foi proposto para criarmos em dupla um jogo 2D em C por meio da biblioteca do RayLib.
Tivemos que fazer um jogo 2D de top-view estilo "The Legend of Zelda" de 96.

---

## 🏗️ Arquitetura e Tecnologias

- C(99) como linguagem principal
- Raylib para renderização 2D, inputs e gerenciamento de janela
- CodeBlocks como ambiente de desenvolvimento
- Arquitetura baseada em Entity(cada objeto representado por uma struct)
- Gerenciamento de recursos, como carregamento de texturas/sons e manipulação de arquivos .txt

---

## 🧱 Boas práticas aplicadas

- Versionamento e fluxo de desenvolvimento por meio do GitHub
- Uso de comentário claros nos códigos
- Separação em .c e .h para manter responsabilidades isoladas
- Testes manuais nos elementos principais do jogo(colisão e movimentação de entidades)

---

## 🤝 Soft Skills e Trabalho em Equipe

- Trabalho colaborativo no GitHub em dupla utilizando branches para garantir a organização. 
- Divisão clara de responsabilidades do sistema de elementos do jogo.  
- Gestão de tempo e definição de prioridades para o avanço das mecânicas.

## 📈 Resultados e Aprendizados
- O que funcionou bem:  
  1. A estrutura modular do código facilitou a divisão do trabalho entre a dupla.
  2. O uso do GitHub ajudou a manter o histórico organizado, principalmente com branches para cada recurso.
  3. A lógica principal (movimentação e colisões) funcionou de forma estável, apesar das correções contínuas de bugs.
  
- O que poderia melhorar:
  1. Organização das pastas (src, sprites, audios), não pensamos muito nisso durante o processo.
  2. Certos algoritmos poderiam ser otimizados com conhecimentos mais avançados atualmente. 
  
- Conceitos mais aplicados da disciplina:  
  1. Programação estruturada e uso de modularização em C, com foco em ponteiros e structs.
  2. Manipulação de arquivos .txt.
  3. Principalmente utilização de arrays para armazenar dados.

- Lições para projetos futuros:  
  1. Iniciar com um design de arquitetura mais claro (diagrama de estados do jogo, como menu e pause).
  2. Estabelecer padrões de código para outros programadores poderem entender desde o início.
